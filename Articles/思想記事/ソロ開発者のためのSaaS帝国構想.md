# ソロ開発者のためのSaaS帝国構想

> 1人で複数プロダクトを運用する戦略

---

## 前提：なぜ複数プロダクトか

「1つのプロダクトに集中すべき」という意見は正しい。**最初は**。

しかし、1つのプロダクトが安定期に入ったら、次を考える必要がある。

- 市場が飽和する可能性
- 収益の分散（リスクヘッジ）
- 既存顧客への追加価値提供

---

## 帝国構想の全体像

```
                    ┌─────────────────┐
                    │  KozoFramework  │
                    │  （共通基盤）     │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
    ┌────▼────┐         ┌────▼────┐         ┌────▼────┐
    │ ZeroBook │         │  Boltry  │         │  ???    │
    │  (予約)   │         │  (TBD)   │         │ (将来)  │
    └────┬────┘         └────┬────┘         └────┬────┘
         │                   │                   │
         └───────────────────┼───────────────────┘
                             │
                    ┌────────▼────────┐
                    │   顧客基盤      │
                    │ （相互送客）     │
                    └─────────────────┘
```

### 3つのレイヤー

1. **共通基盤（KozoFramework）**: 認証、決済、通知、埋め込みなど
2. **個別プロダクト**: ドメイン固有のロジック
3. **顧客基盤**: 既存ユーザーへのクロスセル

---

## 相乗効果（Synergy）

単独で運用するより、複数プロダクトを連携させた方が価値が高まる。

### 例：ZeroBook → Boltry

ZeroBookのユーザーは「予約システム」を使っている。

彼らが次に欲しいのは何か？

- 顧客管理（CRM）
- メール配信
- 分析ダッシュボード

Boltryがこれを提供すれば、既存ユーザーは追加購入する可能性が高い。

---

## Empire Synergy Factor

複数プロダクト展開時の相乗効果を数値化したもの。

| プロダクト数 | 単純合計の価値 | シナジー込みの価値 | ESF |
|-------------|---------------|-------------------|-----|
| 1 | 1.0 | 1.0 | 1.0x |
| 2 | 2.0 | 2.4 | 1.2x |
| 3 | 3.0 | 4.2 | 1.4x |
| 5 | 5.0 | 9.0 | 1.8x |

5プロダクト展開時、単純合計の**1.8倍**の価値を生む。

これは以下の要因による：

- **開発コストの削減**: 共通基盤の再利用（80%）
- **顧客獲得コストの削減**: 既存ユーザーへのクロスセル
- **ブランド力の強化**: 「Kozo製品」としての認知

## 実行戦略

### Phase 1: 基盤確立（現在）
- ZeroBookを主力として安定運用
- KozoFrameworkを抽出・洗練

### Phase 2: 横展開（次）
- Boltry開発着手
- ZeroBookユーザーへの早期案内

### Phase 3: 帝国化（将来）
- 3つ目以降のプロダクト
- 統合ダッシュボード
- 顧客データの横断活用

---

## ソロ開発者にとっての意味

「帝国」という言葉は大げさに聞こえるかもしれない。

しかし、本質は**レバレッジ**だ。

- 1回の開発で、複数プロダクトに反映される
- 1人の顧客が、複数プロダクトを使う
- 1人の開発者が、複数の収益源を持つ

これがソロ開発者の生存戦略。

---

## まとめ

> **「1つを極めた後、横展開で帝国を築く」**

最初から複数をやるのではない。1つを極め、共通基盤を抽出し、それを使って次を作る。

これが「SaaS帝国構想」の本質。
